## CRITICAL Findings (consensus)

### C1 — IEEE 1584 standard is mislabelled (EE)
`backend/analysis/arcflash.py` implements **IEEE 1584-2002** but PDF reports (`pdf_reports.py:536`), arc-flash labels (`:1270`), and the UI claim **"IEEE 1584-2018"**. The 2002 model lacks the 5 electrode configurations, enclosure-size correction, and intermediate-voltage interpolation of 2018. **A user could present 2002 results as 2018-compliant.** Fix: either implement 2018, or correct every label to "IEEE 1584-2002".

### C2 — Native `alert()`/`confirm()`/`prompt()` for core flows (UX)
~13 call sites across `app.js:377`, `project.js:29,518,569,656`, `standard-data.js`, `templates.js`, `properties.js`. Blocking, unstylable, theme-inconsistent, untrappable. Notably `app.js:377` uses `confirm()` to choose fault-scope (selected bus vs all buses) — a blocking dialog mid-analysis with a fragile OK/Cancel semantic.

### C3 — No keyboard navigation for the canvas; modals trap no focus (UX)
SVG canvas has no `tabindex`/`role`; components are unfocusable `<g>` elements; 16+ modals declare `role="dialog"` but no focus trap, no `Escape`-restore-focus. Screen reader users cannot operate the diagram.

### C4 — SELECT-mode port-click silently starts a wire (UX)
`canvas.js:338-346` — clicking a connection port in SELECT mode starts wire drawing with no cursor/banner change and the status bar still reading "Select Mode". Only cue is a subtle `cursor: crosshair` on the port hit-area. Users get stuck mid-wire-draw with no obvious exit except Escape.

### C5 — Revision restore does not clear the undo stack → silent project corruption (DEV)
`revisions.js:232-262` calls `AppState.fromJSON(revData)` but never `UndoManager.clear()`. After restoring revision R, Ctrl+Z replays pre-restore snapshots, then Ctrl+S **overwrites project P with the wrong diagram**. Identical mechanism to the previously-fixed C7 (which was closed for every *other* load path — this one was missed). One-line fix.

## HIGH Findings

### Standards & Calculation (EE)
- **H1 — Single c-factor 1.10 for all voltage levels** (`fault.py:82`). IEC 60909-0 Table 1 requires 1.05 for LV +6% tolerance. Frontend has the data in `constants.js:314-318` but doesn't pass it through. Overstates LV fault current ~4.8%.
- **H2 — Reduced arcing current uses 0.90 for MV** (`arcflash.py:161-164`). IEEE 1584-2002 §5.5 specifies 0.85 at all voltages. Non-standard.
- **H3 — No PV-bus Q-limits in load flow** (`loadflow.py:478-502`). Generators hold |V| with unbounded Q — no Qmin/Qmax enforcement, no PV→PQ conversion. Non-physical reactive output for out-of-capability machines.
- **H4 — Ungrounded generators contribute Z0 to SLG faults** (`fault.py:789-803`). `x0=0` falls back to Z1; for an ungrounded machine Z0 should be ∞. Affects SLG fault current magnitude.
- **H5 — SANS 10142 max-demand sums zero for static_load** (`compliance.js:662-665`). Props key mismatch (`rated_mva` vs `rated_kva`) → static loads contribute nothing to demand.
- **H6 — Missing regression tests** for LLG fault, motor contribution, cable sizing, grounding grid, duty check, load diversity, arc flash boundary. Core fault/SLG/peak/energy/conductor/dip tests exist and pass.
- **H7 — Two cable tables on different temperature bases** (`cable_sizing.py` vs `constants.js`). Internally consistent today but fragile — any merger breaks voltage-drop silently.

### Software / Data Integrity (DEV)
- **H8 — `ProjectData` schema drops groups, pages, analysis results on DB save** (`schemas.py:87-99`). No `extra="allow"`; `model_dump()` strips unknown fields. Save → reopen loses multi-sheet layouts, component groups, and **all result badges**. JSON *export* path is unaffected (uses `AppState.toJSON` directly). Round-trip test needed to confirm.
- **H9 — `Content-Disposition` header injection** (`reports.py:71,100,120`). Unsanitized `projectName` reflected into `filename=`. `\r\n` / `"` enables header splitting. Unauthenticated (no auth).
- **H10 — No payload size limit + no auth + no rate limit** (`analysis.py`, `main.py`). `components: list[Component]` uncapped; dev path has no nginx 10 MB cap. Crafted large project hangs worker threads → backend unresponsive.
- **H11 — CORS `allow_origins=["*"]` + `allow_credentials=True`** (`main.py:18-24`). Spec-illegal combination; Starlette reflects Origin → effectively "any origin with credentials". Latent account-takeover the day auth ships.
- **H12 — SQLite no WAL, no busy_timeout** (`database.py:10`). Auto-save (every 2 min) + manual save / multi-tab → `database is locked` HTTP 500. Two-pragma fix.
- **H13 — Stored project JSON returned unvalidated** (`projects.py:109,171`). Malformed/legacy blobs flow into `AppState.fromJSON` (which trusts shape) → frontend crash on missing `comp.type`. Inconsistent with write path which validates.

### UX
- **H14 — WCAG contrast failures** in `--text-muted: #888` on white (~3.5:1, below AA 4.5:1). Pervasive in status bar, dropdown shortcuts, property titles.
- **H15 — No global analysis busy indicator**. Only the trigger button is disabled; user can keep editing and invalidate in-flight results with no visual lock or spinner.
- **H16 — No toast system; status bar reuses one line** for info/success/error with identical styling. Save *failure* looks like "Copied 3 components".
- **H17 — Toolbar IA: Analysis vs Studies vs Safety split is unclear**; View menu is a flat 14-item grab-bag. Mode (Select/Wire) buried in Edit menu with no persistent on-canvas indicator.
- **H18 — Mobile hides properties panel with no wired replacement** (`mobile-properties-content` appears unpopulated). Mobile also missing Studies, Settings, Scenarios, Revision Timeline, Help.
- **H19 — Annotation badges stack naively** (fixed +70px right column) — overflow into neighbors on dense diagrams; no per-badge collapse/hide.
- **H20 — Branch-flow labels grow to 8+ lines** and overlap adjacent components by default.
- **H21 — Properties non-General sections collapsed by default** — users must click each header to find Fault/LoadFlow/Cable fields.
- **H22 — Export destinations scattered** across File (SVG/PNG/JSON) and Reports (CSV/PDF/labels) with unclear naming; no single "Export" menu.
- **H23 — Dark-mode result-table row colors** (`.af-danger` etc.) have no dark overrides → washed-out pastels on dark modals.
- **H24 — No first-run experience / onboarding tour**; blank canvas on every startup.

## Notable MEDIUM Findings

- **M1** — κ peak factor uses Method A without 1.15 meshed-network factor (`fault.py:1037`) → understates ip on meshed networks.
- **M2** — μ breaking factor uses generator-terminal I"k/Ir, ignores external impedance → understates Ib for remote faults.
- **M3** — q-factor uses I"k/IrM instead of m (MW/pole-pair) per IEC 60909 §9.1.2.
- **M4** — Zero-impedance link modelled as `−j1e6` susceptance → fictitious reactive power in swing-bus Q (`loadflow.py:378`).
- **M5** — Cable ambient derating uses 30°C reference for buried cables (should be 20°C) (`cable_sizing.py:554`).
- **M6** — NEC ampacity lookup allows 20% downsizing (non-conservative) (`cable_sizing.py:256`).
- **M7** — K2 arc-flash grounding factor hardcoded to 0 (ungrounded) → overstates energy for grounded systems.
- **M8** — Motor starting: locked rotor modelled as constant-PQ (pessimistic) vs constant-impedance.
- **M9** — Wire preview is a straight `<line>`, not the orthogonal path that will actually draw (`wiring.js:33`).
- **M10** — Snap-to-port radius is fixed 30 world-px → unusable at 10% zoom, over-eager at 500% (`canvas.js:1351`). Should scale by zoom.
- **M11** — Undo snapshots omit `scenarios` and `annotationOffsets` → partial undo (`undo.js:28`).
- **M12** — `prefers-reduced-motion` not respected; warning rings pulse indefinitely.
- **M13** — No HTTPS, no rate limiting in deployment; `run.sh` ships `--reload` (dev) on `0.0.0.0`.
- **M14** — ~30+ hardcoded hex colors in `app.js` result builders bypass the CSS variable system and break dark mode.
- **M15** — LLG fault reports earth current (3·Ia0) but schema label implies phase-fault current.

## What's Done Well (cross-reviewer consensus)

- **IEC 60909 Z0 network tracing** (`fault.py:748-1011`) — vector-group parsing, delta-blocks-Z0, zigzag-as-source, grounding override. Unusually thorough for a lightweight tool.
- **Newton-Raphson Jacobian** — all 4 sub-matrices (J1–J4) verified term-by-term against standard form.
- **IDMT/IEEE curve constants** and **TCC coordination** (topology-aware pairing, phase/earth separation, voltage referral, fuse I²t rule) — production-grade.
- **Cable k-factors** and **IEEE 80 material constants** match the standards.
- **Prior audit's 8 Critical findings are all fixed** in the working tree, with a standards-anchored regression suite added.
- Mobile UI attempt, file manager, per-unit calc modal, and TCC sub-app exceed typical "web ETAP" scope.
