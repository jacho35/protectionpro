Verification & Validation

ProtectionPro Analysis Engines — Worked Examples

Every one of ProtectionPro's sixteen analysis engines is cross-checked against a published standard, a textbook worked example, or an exact first-principles / hand calculation. This page shows each example as it was worked, the reference figures, ProtectionPro's result, and the source.

← Back to ProtectionPro

How each example was verified

Each reference example is rebuilt as a reproducible ProtectionPro project and solved two ways that must agree to within display rounding: (1) a direct call to the backend analysis engine (authoritative), and (2) the real application driven headlessly, so the on-diagram result badges are genuine renders. The pass criterion is a match to within ±2 % of the reference; anything beyond that is investigated and qualified. Where a case notes a "qualified" result, the governing formulas reproduce the reference exactly and the difference is a documented, more-conservative modelling choice — not a defect.

Short-circuit cases set the IEC 60909 voltage factor c to whatever each reference used (main article c = 1.0; the 220/33 kV article c = 1.10, the ProtectionPro default). Most examples below are available in-app under Templates → Verification / Standards; the stability, contingency and dynamic-motor cases are anchored to the backend regression suite (test_regression.py).
