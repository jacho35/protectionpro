## Overhead Line Model Enhancements
Overhead lines are modelled as a "Feeder Type" of the Cable component (`construction: 'overhead'`), backed by the `STANDARD_OVERHEAD_LINES` codeword library (ACSR/AAAC). They currently feed a pure series R+jX (plus a lumped-earth-return ZŌéĆ) into the same fault/load-flow engines as cables. Fidelity gaps, ranked by impact:

- **Conductor-temperature dependence of R**: the overhead library stores 20 ┬░C resistance and the engines use it as-is, while the underground cable library stores *hot* (90 ┬░C/70 ┬░C) values. In one study, overhead runs cold and cables run hot, so overhead load-flow losses/voltage-drop come out ~15ŌĆō25 % optimistic (Al tempco Ōēł 0.004/┬░C). Add a `temperature_c` prop and apply `R(T) = RŌééŌéĆ┬Ę[1+╬▒(TŌłÆ20)]` so the two libraries are consistent and users can model summer/emergency ratings. *(Smallest, best-bounded fix ŌĆö recommended first.)*
- **Parallel-circuit zero-sequence mutual coupling**: `num_parallel` simply divides ZŌéĆ, but double-circuit lines on shared towers have strong ZŌéĆ mutual coupling that *raises* effective ZŌéĆ and skews ground-fault current. Model the mutual term. *(Highest-value correctness fix; needs a short design pass.)*
- **Geometry-driven reactance**: XŌéü is a single library value at an assumed "typical spacing". Add a spacing/bundle input and compute `XŌéü = 0.145┬ĘlogŌéüŌéĆ(GMD/GMR)` (and support 2/3/4-sub bundle conductors, not representable today) to replace the "adjust for your tower geometry" caveat.
- **Earth-wire / soil-resistivity ZŌéĆ model**: today ZŌéĆ is a fixed Ōēł3.5├Ś multiplier baked into the library with no dependence on soil resistivity or a shield-wire/OPGW return path (which can roughly halve XŌéĆ). Add a Carson-based ZŌéĆ calculator with soil resistivity + optional earth wire.
- **Line charging / shunt susceptance (B)**: both cable and overhead are pure series R+jX. For long HV lines (>~50 km, Ōēź132 kV) shunt capacitance matters for load flow and Ferranti voltage rise. Add a per-km `b_per_km` ŽĆ-model.
- **Dynamic/static thermal rating (ampacity)**: `rated_amps` is one fixed still-air number. Add an IEEE 738 static rating (ambient + wind + solar + conductor temp) with seasonal/emergency ratings and sag-limited limits, since overload flags and cable-sizing verdicts key off this value.
- **Conductor library breadth**: only ACSR + AAAC. Add AAC, ACAR, modern HTLS/ACCC conductors, and copper OH for legacy networks; make AC/skin-effect uplift at 50 Hz explicit for large conductors.
- **Length-default warning**: `length_km` silently defaults to 1 km when absent, which changes every sequence impedance ŌĆö surface a UI warning.
